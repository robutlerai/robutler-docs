    Server Overview

    Deploy agents as production-ready OpenAI-compatible API servers with the Robutler FastAPI server.

    Quick Start

    Basic Server

    from robutler.server import app
from robutler.agents import BaseAgent

# Create your agent
agent = BaseAgent(
    name="my-assistant",
    instructions="You are a helpful assistant",
    model="openai/gpt-4o"
)

# Register with server
app.register_agent(agent)

# Run server
if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000)

    Multiple Agents

    from robutler.server import app
from robutler.agents import BaseAgent
from robutler.agents.skills import (
    ShortTermMemorySkill,
    DiscoverySkill,
    DatabaseSkill
)

# Customer service agent
support_agent = BaseAgent(
    name="customer-support",
    instructions="You are a customer service agent",
    model="openai/gpt-4o",
    skills={
        "memory": ShortTermMemorySkill(),
        "discovery": DiscoverySkill()
    }
)

# Data analysis agent
data_agent = BaseAgent(
    name="data-analyst", 
    instructions="You are a data analysis expert",
    model="anthropic/claude-3-sonnet",
    skills={
        "database": DatabaseSkill(),
        "memory": ShortTermMemorySkill()
    }
)

# Register all agents
app.register_agent(support_agent)
app.register_agent(data_agent)

# Server automatically serves at:
# - /agents/customer-support/chat/completions
# - /agents/data-analyst/chat/completions

    Server Architecture

    FastAPI Application

    The server is built on FastAPI and exposes OpenAI-compatible endpoints per agent. Key features:

    • OpenAI Compatibility - Full /chat/completions API
    • Multi-Agent Routing - /agents/{agent_name}/... endpoints
    • Streaming Support - Server-Sent Events (SSE)
    • Middleware - Authentication, rate limiting, CORS
    • Health Checks - /health endpoint
    • Dynamic Agent Resolution - Load agent configs from the portal on-demand (if enabled)

    Endpoint Structure

    /                          # Root info
/health                    # Health check
/agents                    # List available agents
/agents/{name}/info        # Agent information
/agents/{name}/chat/completions  # OpenAI chat API

    Configuration

    Environment Variables

    # Server configuration
ROBUTLER_HOST=0.0.0.0
ROBUTLER_PORT=8000
ROBUTLER_WORKERS=4

# API keys
OPENAI_API_KEY=your-openai-key
ANTHROPIC_API_KEY=your-anthropic-key

# Database
DATABASE_URL=postgresql://user:pass@localhost/db

# Authentication
JWT_SECRET=your-jwt-secret
ROBUTLER_API_KEY=your-platform-key

    Server Configuration

    from robutler.server import app, ServerConfig

# Configure server
config = ServerConfig(
    host="0.0.0.0",
    port=8000,
    workers=4,
    timeout=60,
    enable_cors=True,
    enable_auth=True,
    rate_limit="100/minute"
)

app.configure(config)

    Authentication & Authorization

    API Key Authentication (via AuthSkill)

    from robutler.server.middleware import AuthMiddleware

# Add auth middleware
auth = AuthMiddleware(
    api_keys=["your-api-key"],
    jwt_secret="your-jwt-secret"
)

app.add_middleware(auth)

    User Context

    When AuthSkill is enabled, identity headers are normalized and propagated on the unified context. Standard headers:

    X-Origin-User-ID
X-Peer-User-ID
X-Agent-Owner-User-ID

    # Access user context in skills
class MySkill(Skill):
    @tool
    def user_specific_action(self, param: str) -> str:
        context = self.get_context()

        user_id = context.peer_user_id
        auth_scope = context.get("auth_scope", "user")

        if auth_scope == "admin":
            return f"Admin action for {user_id}: {param}"
        else:
            return f"User action for {user_id}: {param}"

    Custom HTTP Endpoints

    Agents can provide custom HTTP endpoints using the @http decorator. These endpoints are automatically registered with the FastAPI server when the agent is added.

    Basic HTTP Endpoints

    from robutler.agents.tools.decorators import http
from robutler.agents import BaseAgent

@http("/weather", method="get", scope="owner")
def get_weather(location: str, units: str = "celsius") -> dict:
    """Weather API endpoint"""
    return {
        "location": location,
        "temperature": 25,
        "units": units,
        "condition": "sunny"
    }

@http("/data", method="post")
async def post_data(data: dict) -> dict:
    """Data submission endpoint"""
    return {
        "received": data,
        "status": "processed",
        "timestamp": "2024-01-01T12:00:00Z"
    }

# Create agent with HTTP endpoints
agent = BaseAgent(
    name="api-agent",
    model="openai/gpt-4o",
    http_handlers=[get_weather, post_data]
)

# Register with server
app.register_agent(agent)

    Available Endpoints

    Once registered, the agent's HTTP endpoints will be available at:

    • Core Endpoints:
    • GET /api-agent - Agent information

    • POST /api-agent/chat/completions - OpenAI-compatible chat

    • Custom Endpoints:

    • GET /api-agent/weather?location=NYC&units=fahrenheit
    • POST /api-agent/data (with JSON body)

    HTTP Method Support

    All standard HTTP methods are supported:

    @http("/resource", method="get")      # Read
@http("/resource", method="post")     # Create
@http("/resource", method="put")      # Update/Replace
@http("/resource", method="patch")    # Partial Update
@http("/resource", method="delete")   # Delete
@http("/resource", method="head")     # Headers Only
@http("/resource", method="options")  # CORS Preflight

    Scope-Based Access Control

    HTTP endpoints support scope-based access control:

    @http("/public", scope="all")         # Anyone can access
@http("/user-data", scope="owner")    # Agent owner only
@http("/admin", scope="admin")        # Admin users only
@http("/multi", scope=["owner", "admin"])  # Multiple scopes

    Path Parameters and Query Strings

    HTTP endpoints automatically receive FastAPI request parameters:

    @http("/files/{file_id}")
def get_file(file_id: str, download: bool = False) -> dict:
    """File endpoint with path parameter and query parameter"""
    return {
        "file_id": file_id,
        "filename": f"document_{file_id}.pdf",
        "download": download
    }

# Available as: GET /api-agent/files/123?download=true

    JSON Request Bodies

    For POST/PUT/PATCH requests, JSON data is automatically parsed:

    @http("/process", method="post")
async def process_data(data: dict, priority: str = "normal") -> dict:
    """Process data with optional priority query parameter"""
    return {
        "data": data,           # From JSON body
        "priority": priority,   # From query parameter
        "status": "processed"
    }

# Usage: POST /api-agent/process?priority=high
# Body: {"message": "Hello", "value": 42}

    Error Handling

    HTTP endpoints automatically handle errors and return appropriate HTTP status codes:

    @http("/divide")
def divide_numbers(a: float, b: float) -> dict:
    """Division endpoint with error handling"""
    if b == 0:
        raise ValueError("Division by zero not allowed")

    return {
        "result": a / b,
        "operation": f"{a} / {b}"
    }

# Returns HTTP 500 with error message for b=0

    Integration with Agent Tools

    HTTP endpoints work alongside regular agent tools:

    from robutler.agents.tools.decorators import tool, http

@tool(scope="owner")
def calculate(expression: str) -> str:
    """Internal calculation tool"""
    return str(eval(expression))

@http("/calc")
def calc_api(expression: str) -> dict:
    """HTTP endpoint that uses the internal tool"""
    result = calculate(expression)
    return {
        "expression": expression,
        "result": result,
        "type": "calculation"
    }

agent = BaseAgent(
    name="calc-agent",
    tools=[calculate],           # Internal tool for LLM
    http_handlers=[calc_api]     # External HTTP API
)

    Client Usage

    OpenAI SDK Compatible

    import openai

# Configure client
client = openai.OpenAI(
    base_url="http://localhost:8000/agents/my-assistant",
    api_key="your-api-key"
)

# Standard chat completion
response = client.chat.completions.create(
    model="gpt-4o",  # Model is handled by agent
    messages=[
        {"role": "user", "content": "Hello!"}
    ]
)

print(response.choices[0].message.content)

    Streaming

    # Streaming response
stream = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "user", "content": "Tell me a story"}
    ],
    stream=True
)

for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="")

    Tool Calling

    # External tools
tools = [{
    "type": "function",
    "function": {
        "name": "get_weather",
        "description": "Get weather for location",
        "parameters": {
            "type": "object",
            "properties": {
                "location": {"type": "string"}
            },
            "required": ["location"]
        }
    }
}]

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "user", "content": "What's the weather in Tokyo?"}
    ],
    tools=tools
)

# Handle tool calls
message = response.choices[0].message
if message.tool_calls:
    # Execute tools and continue conversation
    pass

    Production Deployment

    Docker

    FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install -r requirements.txt

COPY . .

EXPOSE 8000

CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]

    # docker-compose.yml
version: '3.8'

services:
  robutler:
    build: .
    ports:
      - "8000:8000"
    environment:
      - OPENAI_API_KEY=${OPENAI_API_KEY}
      - DATABASE_URL=postgresql://postgres:password@db:5432/robutler
    depends_on:
      - db

  db:
    image: postgres:15
    environment:
      - POSTGRES_DB=robutler
      - POSTGRES_PASSWORD=password
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:

    Kubernetes

    # deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: robutler-server
spec:
  replicas: 3
  selector:
    matchLabels:
      app: robutler-server
  template:
    metadata:
      labels:
        app: robutler-server
    spec:
      containers:
      - name: robutler
        image: your-registry/robutler:latest
        ports:
        - containerPort: 8000
        env:
        - name: OPENAI_API_KEY
          valueFrom:
            secretKeyRef:
              name: robutler-secrets
              key: openai-api-key
        resources:
          requests:
            memory: "512Mi"
            cpu: "250m"
          limits:
            memory: "1Gi"
            cpu: "500m"

---
apiVersion: v1
kind: Service
metadata:
  name: robutler-service
spec:
  selector:
    app: robutler-server
  ports:
  - protocol: TCP
    port: 80
    targetPort: 8000
  type: LoadBalancer
